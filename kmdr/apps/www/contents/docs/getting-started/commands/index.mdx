---
title: Commands
description: Kommander is a flexible CLI tool for executing and verifying remote configurations, allowing developers to create easy, quick, and reusable configurations for various developer tools.
---

There are 3 primary commands provided by Kommander

1. <a href="/docs/commands/init">`init`</a> to setup a `setup.json` file for you
   to store state and context of a project. By default, this will prompt the user
   for the `targetSrc` where all the Kommander related files will go and the package
   manager the user is currently using.

2. <a href="/docs/commands/run">`run`</a> to fetch and executes remote configurations,

3. <a href="/docs/commands/verify">`verify`</a>: Validates if the remote
   configurations matches specifications.
